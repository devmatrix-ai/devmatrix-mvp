# Frontend Accessibility

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle frontend accessibility.

## When to use this skill:

- When creating or editing UI component files (e.g., `components/*.tsx`, `components/*.jsx`, `*.vue`, `*.svelte`)
- When implementing keyboard navigation and focus management in interactive elements
- When adding ARIA attributes (aria-label, aria-describedby, role) to enhance screen reader support
- When ensuring proper color contrast ratios meet WCAG AA or AAA standards
- When providing alternative text for images, icons, and visual content
- When creating forms with proper label associations and error messaging
- When building modal dialogs, dropdowns, or custom interactive widgets
- When establishing semantic HTML structure with proper heading levels (h1-h6)
- When testing components with screen readers (NVDA, JAWS, VoiceOver)
