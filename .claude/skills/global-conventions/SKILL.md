# Global Conventions

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle global conventions.

## When to use this skill:

- When organizing or restructuring project files and directory layouts
- When writing or updating README.md files with setup instructions and architecture overviews
- When creating git commits with clear messages or working with feature branches
- When configuring environment variables and ensuring secrets are not committed to version control
- When managing project dependencies in package.json, requirements.txt, or similar files
- When documenting why major dependencies are used in the project
- When establishing or following code review processes and expectations
- When implementing feature flags for incomplete features
- When maintaining CHANGELOG.md or release notes for tracking project changes
- When creating or updating contribution guidelines and development workflow documentation
